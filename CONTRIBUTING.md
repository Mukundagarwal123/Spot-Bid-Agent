# Contributing

## Branching
- Use feature branches from `main`.
- Keep PRs focused and small.

## Code Standards
- Python typing required for public functions.
- Structured logging required for all business-critical handlers.
- Add tests for new behavior and bug fixes.

## Commit Guidance
- `feat: ...` for features
- `fix: ...` for bug fixes
- `docs: ...` for documentation updates

## Definition of Done
1. Tests pass.
2. Logs include correlation fields.
3. No secrets committed.
4. Spec/doc updates included for behavior changes.
